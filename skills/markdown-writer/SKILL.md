# Markdown Writer

Write Markdown for humans, not for the agent's ego.

## Default workflow

1. Identify the document type and audience.
2. Start with the simplest structure that fits.
3. Use headings, lists, code blocks, and tables only when they improve scanning.
4. Keep examples runnable and specific.

## Strong defaults

- lead with the most useful thing
- keep sections short
- avoid filler prose
- use tables for genuinely tabular content
- include examples where action matters

## Good outputs

- README
- how-to guide
- technical reference
- changelog
- setup guide
- feature overview

## Caution

Do not confuse “long” with “documented.” A bloated README is just a slower disappointment.
